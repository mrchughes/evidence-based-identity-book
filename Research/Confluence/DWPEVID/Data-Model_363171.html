DCS Evidence-Based Identity Platform : Data Model

Created by Christian Hughes, last modified on Sept 15, 2025

Core RDF Triple Model

The evidence-based identity platform represents all information as Resource Description Framework (RDF) triples following the fundamental pattern of subject-predicate-object statements with comprehensive metadata about provenance, confidence, and temporal validity. This uniform representation enables sophisticated reasoning capabilities while maintaining complete audit trails for every piece of information in the system.

Every assertion in the platform follows the basic triple structure where a subject (who or what the assertion is about) is connected to an object (the value or entity being asserted) through a predicate (the relationship or property). For example, the assertion "John Smith earns Â£30,000 annually" becomes the triple subject:john_smith_cluster_123 dcs:hasAnnualIncome "30000"^^xsd:decimal with additional metadata triples describing confidence, source, and verification details.

The model extends basic RDF triples with comprehensive provenance metadata that captures who made each assertion, when it was made, how it was verified, and what confidence level should be assigned. This metadata exists as additional triples that reference the primary assertion, creating a rich web of interconnected information that supports both operational decision-making and comprehensive audit requirements.

# Primary assertion
:john_smith_cluster_123 dcs:hasAnnualIncome "30000"^^xsd:decimal .

# Provenance metadata
:assertion_456 rdf:type prov:Entity ;
    prov:wasAttributedTo :revenue_authority_system ;
    prov:generatedAtTime "2025-09-15T10:30:00Z"^^xsd:dateTime ;
    dcs:hasConfidence "0.85"^^xsd:decimal ;
    dcs:verificationMethod "employer_digital_submission" ;
    dcs:aboutSubject :john_smith_cluster_123 ;
    dcs:assertsProperty dcs:hasAnnualIncome ;
    dcs:assertsValue "30000"^^xsd:decimal .

Identity Cluster Data Structure

Identity clusters represent the platform's solution to the fundamental challenge of correlating evidence about the same real-world person across different systems and contexts. Each cluster maintains probabilistic relationships between subject identifiers while preserving uncertainty through confidence scoring.

The cluster structure includes references to all subject identifiers that likely represent the same person, confidence scores for the overall cluster and individual correlations, temporal information about when correlations were established, and metadata about the clustering algorithms and evidence that supported correlation decisions.

:identity_cluster_789 rdf:type dcs:IdentityCluster ;
    dcs:hasOverallConfidence "0.85"^^xsd:decimal ;
    dcs:containsSubject :subject_nino_AB123456C ;
    dcs:containsSubject :subject_did_citizen123 ;
    dcs:containsSubject :subject_la_housing_456 ;
    dcs:clusteringAlgorithm "fuzzy_match_v2.1" ;
    dcs:lastUpdated "2025-09-15T14:20:00Z"^^xsd:dateTime ;
    dcs:createdBy :identity_resolution_service .

# Individual correlation confidence
:correlation_123 rdf:type dcs:SubjectCorrelation ;
    dcs:correlatesSubject :subject_nino_AB123456C ;
    dcs:correlatesSubject :subject_did_citizen123 ;
    dcs:correlationConfidence "0.92"^^xsd:decimal ;
    dcs:basedOnEvidence :shared_name_evidence, :shared_address_evidence ;
    dcs:correlationMethod "attribute_overlap_analysis" .

Cluster evolution tracking maintains historical information about how clusters have changed over time, including merge operations when new evidence suggests separate identities are the same person, split operations when investigation reveals incorrect merging, and confidence adjustments based on evidence quality changes.

Evidence Assertion Model

Evidence assertions capture structured claims about citizen circumstances with complete context about verification methods, source credibility, and confidence assessment. The assertion model accommodates diverse evidence types while maintaining consistent metadata patterns that enable cross-evidence reasoning and correlation.

Each assertion includes the core claim being made, attribution to the asserting organization or system, verification method details that explain how the claim was validated, confidence scoring based on verification strength and source credibility, temporal validity indicating when the assertion applies and when it expires, and supporting evidence references that provide additional context.

:assertion_employment_123 rdf:type dcs:EmploymentAssertion ;
    dcs:aboutSubject :subject_nino_AB123456C ;
    dcs:hasEmployer "TechCorp Solutions Ltd" ;
    dcs:hasAnnualSalary "30000"^^xsd:decimal ;
    dcs:employmentStartDate "2025-01-15"^^xsd:date ;
    prov:wasAttributedTo :revenue_authority_system ;
    dcs:verificationMethod "employer_digital_filing" ;
    dcs:hasConfidence "0.90"^^xsd:decimal ;
    dcs:effectiveFrom "2025-01-15"^^xsd:date ;
    dcs:validUntil "2026-01-15"^^xsd:date ;
    dcs:supportingDocument :payslip_scan_456 .

:assertion_address_124 rdf:type dcs:AddressAssertion ;
    dcs:aboutSubject :subject_nino_AB123456C ;
    dcs:hasAddress "123 Oak Street, London, SW1A 1AA" ;
    dcs:hasPostcode "SW1A 1AA" ;
    dcs:addressType "residential" ;
    prov:wasAttributedTo :council_tax_system ;
    dcs:verificationMethod "council_tax_record" ;
    dcs:hasConfidence "0.75"^^xsd:decimal ;
    dcs:effectiveFrom "2024-04-01"^^xsd:date .

The assertion model supports hierarchical evidence structures where complex evidence items can be decomposed into multiple related assertions. A payslip document generates assertions about employment, income, tax status, and payment frequency, with relationships between assertions maintained through supporting evidence links.

Confidence and Uncertainty Representation

Confidence scoring represents the platform's explicit acknowledgment that government decisions must be made with incomplete information while providing transparent frameworks for managing uncertainty. The confidence model operates at multiple levels with different scoring algorithms appropriate for each type of uncertainty.

Evidence-level confidence reflects the reliability of individual assertions based on source credibility, verification method strength, document quality, and temporal factors. Identity-level confidence assesses how certain the system is about identity clustering decisions, accounting for correlation strength and evidence consistency. Decision-level confidence combines evidence and identity confidence with policy-specific requirements to determine overall reliability for specific benefit decisions.

:confidence_assessment_123 rdf:type dcs:ConfidenceAssessment ;
    dcs:assessesAssertion :assertion_employment_123 ;
    dcs:overallConfidence "0.85"^^xsd:decimal ;
    dcs:sourceCredibility "0.90"^^xsd:decimal ;
    dcs:verificationStrength "0.88"^^xsd:decimal ;
    dcs:temporalReliability "0.82"^^xsd:decimal ;
    dcs:corroborationLevel "0.80"^^xsd:decimal ;
    dcs:calculatedBy :confidence_scoring_algorithm_v2_1 ;
    dcs:calculatedAt "2025-09-15T10:30:00Z"^^xsd:dateTime .

:identity_confidence_456 rdf:type dcs:IdentityConfidence ;
    dcs:assessesCluster :identity_cluster_789 ;
    dcs:overallConfidence "0.85"^^xsd:decimal ;
    dcs:attributeConsistency "0.88"^^xsd:decimal ;
    dcs:temporalStability "0.82"^^xsd:decimal ;
    dcs:crossValidation "0.87"^^xsd:decimal ;
    dcs:evidenceQuality "0.83"^^xsd:decimal .

Confidence decay models account for temporal factors that affect evidence reliability over time. Employment evidence from recent payslips receives higher confidence than historical salary information, while address verification loses confidence unless refreshed through recent interaction or external confirmation.

Semantic Translation and Ontology Model

The semantic translation layer enables evidence sharing between different policy contexts through formal ontology mappings that preserve meaning while accommodating legitimate vocabulary differences between DCS products and external partners.

The ontology model includes core DCS vocabularies that define common concepts across all benefit areas, product-specific vocabularies for BasicSupport, FamilyAssist, AgeCare and other services, external partner vocabularies for Revenue Authority, local authorities, and healthcare providers, and formal semantic mappings that define equivalence and transformation rules between different vocabulary systems.

dcs-bs:monthlyIncome owl:equivalentProperty dcs-ac:weeklyIncome .
dcs-bs:monthlyIncome dcs:conversionFactor "4.33"^^xsd:decimal .
revenue:annualSalary dcs:equivalentTo dcs-bs:monthlyIncome .
revenue:annualSalary dcs:conversionFormula "annual Ã· 12" .

# Semantic translation result
:translated_assertion_789 rdf:type dcs:TranslatedAssertion ;
    dcs:sourceAssertion :assertion_employment_123 ;
    dcs:sourceContext "revenue_authority_vocabulary" ;
    dcs:targetContext "basic_support_vocabulary" ;
    dcs:translatedProperty dcs-bs:monthlyIncome ;
    dcs:translatedValue "2500.00"^^xsd:decimal ;
    dcs:translationConfidence "0.83"^^xsd:decimal ;
    dcs:translationMethod "annual_to_monthly_conversion" ;
    dcs:appliedMapping :mapping_annual_to_monthly .

Reasoning capabilities enable derivation of new assertions from existing evidence through logical inference rules defined in the ontologies. Family relationship evidence combined with individual income assertions can derive household income calculations relevant for means-tested benefits.

Award and Decision Model

Award calculations represent the culmination of evidence-based processing where verified information about citizen circumstances is converted into specific benefit entitlements through policy rule application and confidence threshold management.

The award model captures entitlement decisions with complete audit trails linking back to supporting evidence, policy rules applied, confidence assessments, and any human interventions or overrides. Each award maintains references to the identity cluster it benefits, the evidence that supported the calculation, confidence levels for both identity and attributes, policy rules and rates applied, effective dates and review schedules, and audit trails for all changes and recalculations.

:award_bs_123456 rdf:type dcs:BasicSupportAward ;
    dcs:beneficiary :identity_cluster_789 ;
    dcs:weeklyAmount "180.50"^^xsd:decimal ;
    dcs:calculationDate "2025-09-15T10:00:00Z"^^xsd:dateTime ;
    dcs:effectiveFrom "2025-09-15"^^xsd:date ;
    dcs:reviewDate "2026-03-15"^^xsd:date ;
    dcs:identityConfidence "0.85"^^xsd:decimal ;
    dcs:attributeConfidence "0.80"^^xsd:decimal ;
    dcs:overallDecisionConfidence "0.82"^^xsd:decimal ;
    dcs:status dcs:active .

# Award calculation breakdown

Historical award tracking maintains complete records of award changes over time, enabling retrospective analysis and supporting appeals processes where citizens need to understand how decisions evolved based on changing evidence.

Investigation and Workflow Model

Investigation cases represent structured workflows for handling exceptions when confidence levels are insufficient for automated processing or when evidence conflicts require human intervention. The investigation model supports complex case management while maintaining comprehensive audit trails.

Investigation entities include case identification and prioritization information, workflow state tracking through defined stages, evidence correlation and conflict analysis, caseworker assignment and skill matching, outcome recording and follow-up actions, and integration with award calculation for decision implementation.

:investigation_789 rdf:type dcs:Investigation ;
    dcs:investigationType dcs:identityVerification ;
    dcs:subject :identity_cluster_789 ;
    dcs:priority dcs:high ;
    dcs:createdDate "2025-09-15T14:00:00Z"^^xsd:dateTime ;
    dcs:triggerReason "confidence_below_threshold" ;
    dcs:triggerEvidence :assertion_conflict_456 ;
    dcs:assignedTo :caseworker_123 ;
    dcs:currentStage "evidence_review" ;
    dcs:estimatedResolution "2025-09-17T17:00:00Z"^^xsd:dateTime .

# Investigation outcome

    dcs:forInvestigation :investigation_789 ;
    dcs:resolution "identity_verified" ;
    dcs:identityConfidenceUpdate "0.90"^^xsd:decimal ;

Workflow state management through http://Temporal.io integration enables complex investigation processes with state persistence, automatic escalation, and recovery capabilities when investigations span multiple days or require coordination between different teams.

External Integration Data Model

External partner integration requires careful data modeling that accommodates diverse partner systems while maintaining semantic consistency and audit requirements. The integration model handles evidence submission from external sources, status tracking and callback management, data quality assessment and validation, and semantic translation for cross-organizational evidence sharing.

Partner submissions include complete provenance information about external systems, verification methods used by submitting organizations, data quality scores and validation results, and semantic context for appropriate evidence interpretation within DCS systems.

:external_submission_123 rdf:type dcs:ExternalSubmission ;

Integration audit trails maintain complete records of all external interactions, enabling comprehensive analysis of partner data quality, processing efficiency, and decision impact while supporting regulatory compliance and continuous improvement initiatives.
